ESRF.net Validation Lab

A reusable hidden environment for validating future ESRF.net pages, forms and features before they are promoted to the public site. Each module is owned, dated, and has explicit exit criteria.

Rules

  1. Every module page carries noindex,nofollow meta tags for both general bots and Googlebot.
  2. Modules are never linked from public navigation, footer, sitemap, robots.txt, news data, editorials list, index, or submit pages.
  3. Test data only — no real sensitive or operational information.
  4. Each module must have an owner, purpose, lastUpdated date, and exit criteria registered in validation-lab.json.
  5. Promotion to main requires explicit reviewer approval on the draft PR. The test branch alone is never auto-merged.

Pre-promotion checklist

  • ☐ noindex,nofollow present on every new module page
  • ☐ No links from public pages, sitemap, robots, navigation or footer
  • ☐ Test/validation banner visible on every module page
  • ☐ Owner and exit criteria recorded in validation-lab.json
  • ☐ Test data only — no sensitive or operational data entered
  • ☐ Reviewed on the Cloudflare branch preview, not on production
  • ☐ Draft PR reviewed and explicitly approved before any promotion to main
